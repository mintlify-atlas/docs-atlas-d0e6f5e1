## Welcome to Laravel Films & Actors

This quickstart guide will help you clone the repository, set up the database, and start exploring the application with real film and actor data.

<Note>
  This application requires PHP 8.1+, Composer, and MySQL to run locally.
</Note>

## Quick Setup

<Steps>
  <Step title="Clone the Repository">
    Clone the Laravel Films & Actors repository to your local machine:

    ```bash
    git clone <repository-url>
    cd laravel-films-actors
    ```
  </Step>

  <Step title="Install Dependencies">
    Install PHP dependencies using Composer:

    ```bash
    composer install
    ```

    This will install:
    - Laravel Framework 10.x
    - Laravel Sanctum for API authentication
    - Doctrine DBAL for database abstraction
    - Development tools (PHPUnit, Laravel Pint, Faker)
  </Step>

  <Step title="Configure Environment">
    Copy the example environment file and generate an application key:

    ```bash
    cp .env.example .env
    php artisan key:generate
    ```

    Update your `.env` file with database credentials:

    ```bash
    DB_CONNECTION=mysql
    DB_HOST=127.0.0.1
    DB_PORT=3306
    DB_DATABASE=laravel
    DB_USERNAME=root
    DB_PASSWORD=
    ```

    <Tip>
      Make sure your MySQL server is running before proceeding to the next step.
    </Tip>
  </Step>

  <Step title="Set Up Database">
    Create the database and run migrations:

    ```bash
    # Create database (if not exists)
    mysql -u root -e "CREATE DATABASE IF NOT EXISTS laravel;"

    # Run migrations
    php artisan migrate
    ```

    This creates three main tables:
    - `films` - Stores film information (name, year, genre, country, duration, image URL)
    - `actors` - Stores actor information (name, surname, birthdate, country, agency, image URL)
    - `films_actor` - Pivot table linking films and actors (many-to-many relationship)
  </Step>

  <Step title="Seed Sample Data">
    Populate the database with sample films and actors:

    ```bash
    php artisan db:seed
    ```

    This command runs the following seeders:
    - **ActorFakeSeeder** - Creates 10 actors with realistic data using Faker
    - **FilmFakeSeeder** - Creates 10 films across various genres (Acción, Drama, Comedia, Terror, Sci-Fi)
    - **FilmsActorSeeder** - Creates relationships between films and actors

    <Note>
      The seeder uses Faker to generate realistic names, countries, and other data. Each run will produce different sample data.
    </Note>
  </Step>

  <Step title="Start the Development Server">
    Launch the Laravel development server:

    ```bash
    php artisan serve
    ```

    Your application will be available at `http://localhost:8000`
  </Step>
</Steps>

## Example Routes to Try

The application comes with several pre-configured routes you can test immediately:

### Film Routes

```bash
# List films before a specific year
http://localhost:8000/filmout/oldFilms/1990

# List films after a specific year
http://localhost:8000/filmout/newFilms/2000

# List films by year
http://localhost:8000/filmout/filmsByYear/2020

# List films by genre
http://localhost:8000/filmout/filmsByGenre/Drama

# Count all films
http://localhost:8000/filmout/countFilms

# View films sorted by year (newest first)
http://localhost:8000/filmout/sortFilms
```

### Actor Routes

```bash
# List all actors
http://localhost:8000/allActors

# List actors by decade
http://localhost:8000/actorsByDecade/1980
```

<Tip>
  All routes with the `/filmout` prefix are protected by a custom year validation middleware that ensures year parameters are within valid ranges.
</Tip>

## Troubleshooting

<AccordionGroup>
  <Accordion title="Database connection error">
    Ensure your MySQL server is running and credentials in `.env` are correct:

    ```bash
    # Test MySQL connection
    mysql -u root -p
    ```

    Verify the database exists:
    ```sql
    SHOW DATABASES LIKE 'laravel';
    ```
  </Accordion>

  <Accordion title="Migration fails">
    If migrations fail, try resetting the database:

    ```bash
    php artisan migrate:fresh --seed
    ```

    <Warning>
      This command drops all tables and re-runs migrations. Only use in development!
    </Warning>
  </Accordion>

  <Accordion title="Composer install errors">
    Make sure you have the required PHP version:

    ```bash
    php -v  # Should be 8.1 or higher
    ```

    Clear Composer cache and retry:
    ```bash
    composer clear-cache
    composer install
    ```
  </Accordion>

  <Accordion title="Port 8000 already in use">
    Specify a different port when starting the server:

    ```bash
    php artisan serve --port=8080
    ```
  </Accordion>
</AccordionGroup>
