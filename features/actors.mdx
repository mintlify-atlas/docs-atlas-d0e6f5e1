## Overview

The Actor Management system uses Eloquent ORM to manage actor data stored in a MySQL database. Unlike films (which use JSON), actors are stored in a relational database with support for filtering by birth decade.

<Note>
Actors use Laravel's Eloquent ORM with the `Actor` model. All queries are sorted by birthdate in ascending order by default.
</Note>

## Actor Model

The `Actor` model defines the database structure and fillable fields:

```php Actor.php
namespace App\Models;

use Illuminate\Database\Eloquent\Factories\HasFactory;
use Illuminate\Database\Eloquent\Model;

class Actor extends Model
{
    use HasFactory;

    protected $fillable = [
        'name',
        'surname',
        'birthdate',
        'country',
        'agency',
        'img_url',
    ];
}
```

## Core Methods

### Reading All Actors

The `readActors()` static method retrieves all actors from the database, ordered by birthdate:

```php ActorController.php
public static function readActors(): array
{
    $actors = Actor::orderBy('birthdate', 'asc')->get()->toArray();

    return is_array($actors) ? $actors : [];
}
```

<Note>
This method returns actors as an array sorted chronologically by birth year, making it easy to display actors from oldest to youngest.
</Note>

### Listing All Actors

Display all actors with decade filter options:

```php ActorController.php
public function listAllActors()
{
    $title = "Listado de todos los actores";
   
    $actors = ActorController::readActors();

    if (empty($actors)) {
        $title = "No hay actores disponibles";
        return view("actors.list", [
            "actors" => [],
            "title" => $title
        ]);
    }

    return view("actors.list", [
        "actors" => $actors,
        "title" => $title,
        "decades" => ActorController::getAvailableDecades(),
        "selectedDecade" => null
    ]);
}
```

**Route:**
```php
Route::get('allActors', [ActorController::class, "listAllActors"])
    ->name('allActors');
```

**URL:** `/allActors`

## Decade Filtering

### Getting Available Decades

The system automatically calculates which decades have actors:

```php ActorController.php
public static function getAvailableDecades(): array
{
    $years = Actor::select('birthdate')
        ->distinct()
        ->orderBy('birthdate', 'asc')
        ->pluck('birthdate')
        ->toArray();
    
    $decades = [];
    foreach ($years as $y) {
        $decade = floor($y / 10) * 10;
        if (!in_array($decade, $decades)) {
            $decades[] = (int)$decade;
        }
    }
    return $decades;
}
```

<Accordion title="How Decade Calculation Works">
  The method:
  1. Queries all unique birth years from the database
  2. Calculates the decade for each year using `floor($year / 10) * 10`
  3. Removes duplicates to get unique decades
  4. Returns sorted array of decades (e.g., [1950, 1960, 1970, 1980])

  **Example:**
  - Birth year `1985` → Decade `1980` (floor(1985/10) * 10 = 198 * 10)
  - Birth year `1992` → Decade `1990`
  - Birth year `1960` → Decade `1960`
</Accordion>

### Filtering by Decade

Filter actors born in a specific decade (10-year range):

```php ActorController.php
public function listActorsByDecade($year = null)
{
    $decades = ActorController::getAvailableDecades();

    if (is_null($year)) {
        $title = "Listado de todos los actores (sin filtrar por década)";
        $actors = ActorController::readActors();
    } else {
        $endYear = $year + 9;
        $title = "Listado de actores nacidos en la década de $year a $endYear";
        $actors = Actor::whereBetween('birthdate', [$year, $endYear])
            ->orderBy('birthdate', 'asc')
            ->get()
            ->toArray();
    }

    return view("actors.list", [
        "actors" => $actors, 
        "title" => $title, 
        "decades" => $decades,
        "selectedDecade" => $year
    ]);
}
```

**Route:**
```php
Route::get('actorsByDecade/{year?}', [ActorController::class, "listActorsByDecade"])
    ->name('actorsByDecade');
```

**URL Examples:**
- `/actorsByDecade` - All actors (no filter)
- `/actorsByDecade/1980` - Actors born 1980-1989
- `/actorsByDecade/1970` - Actors born 1970-1979

<Tabs>
  <Tab title="Query Explanation">
    The decade filter uses Laravel's `whereBetween` to query a range:

    ```php
    Actor::whereBetween('birthdate', [$year, $endYear])
    ```

    - `$year` is the decade start (e.g., 1980)
    - `$endYear` is `$year + 9` (e.g., 1989)
    - Includes both boundary years
  </Tab>

  <Tab title="SQL Equivalent">
    The Eloquent query translates to:

    ```sql
    SELECT * FROM actors 
    WHERE birthdate BETWEEN 1980 AND 1989 
    ORDER BY birthdate ASC
    ```
  </Tab>
</Tabs>

## Route Protection

Actor routes are protected by the `ValidateURLActor` middleware:

```php routes/web.php
Route::middleware('actor')->group(function() {
    Route::get('allActors', [ActorController::class, "listAllActors"])
        ->name('allActors');
    Route::get('actorsByDecade/{year?}', [ActorController::class, "listActorsByDecade"])
        ->name('actorsByDecade');
});
```

The middleware validates year parameters to prevent invalid requests. See [Filtering](/features/filtering) for details.

## Database vs JSON Storage

<Warning>
Actors use **database storage** (Eloquent ORM), while films use **JSON file storage**. This is an important architectural difference.
</Warning>

### Comparison

| Feature | Actors | Films |
|---------|--------|-------|
| **Storage** | MySQL Database | JSON File |
| **ORM** | Eloquent Model | Direct file reading |
| **Querying** | `Actor::where()` | Manual array filtering |
| **Sorting** | `->orderBy()` | `usort()` |
| **Filtering** | `whereBetween()` | `foreach` loops |
| **Performance** | Optimized queries | Full file reads |
